# Unity FPS 热更新和登录系统实现计划

## 背景

用户的Unity FPS项目目前是一个单机原型，包含基础的移动、视角和射击功能。现在需要添加：
1. **资源热更新系统** - 支持美术资源、配置文件、预制件的在线更新
2. **服务器登录系统** - 支持游客登录、账号注册登录、找回密码、记住登录状态

项目当前状态：
- Unity 2022.3.62f2c1，使用URP渲染管线
- 无网络功能、无资源管理系统、无数据持久化
- 现有脚本：PlayerMovement.cs, MouseMovement.cs, Weapon.cs, Bullet.cs

## 技术方案

### 热更新：Addressables
- Unity官方推荐方案，内置版本管理和差异更新
- 自动处理资源依赖，比传统AssetBundle更易用
- 支持本地和远程资源无缝切换

### 登录系统：UnityWebRequest + RESTful API
- Unity原生网络支持，轻量级
- 密码使用SHA256哈希传输
- Token使用JWT标准，AES加密本地存储

## 关键文件职责

### 核心管理器
- **GameManager.cs** - 游戏总控，管理启动流程和状态机（Initializing → CheckingUpdate → Downloading → Login → InGame）
- **NetworkManager.cs** - 封装UnityWebRequest，提供GET/POST方法，处理超时和重试
- **ConfigManager.cs** - 读取ServerConfig.json和AppVersion.json

### 热更新系统
- **HotUpdateManager.cs** - 协调版本检查和资源下载流程
- **VersionChecker.cs** - 请求服务器版本API，比较本地和远程版本号
- **ResourceDownloader.cs** - 使用Addressables下载资源，提供进度回调
- **AddressableLoader.cs** - 封装Addressables资源加载API

### 登录系统
- **LoginManager.cs** - 处理游客登录、账号登录、注册、找回密码逻辑
- **AuthService.cs** - 封装登录/注册/验证API调用
- **TokenManager.cs** - Token的加密存储、读取、验证
- **UserData.cs** - 用户数据模型（userId, username, email, token, isGuest等）

### 工具类
- **Crypto.cs** - SHA256密码哈希、AES加密解密
- **JsonHelper.cs** - JSON序列化/反序列化辅助
- **Logger.cs** - 统一日志输出

## 服务器API设计

### 热更新API
```
GET /api/version/check?platform=windows&currentVersion=1.0.0
Response: { success, data: { version, buildNumber, catalogUrl, totalSize, forceUpdate, updateDescription } }

GET /api/resources/catalog?version=1.0.1
Response: Addressables catalog JSON
```

### 登录认证API
```
POST /api/auth/guest
Body: { deviceId }
Response: { success, data: { userId, username, token, tokenExpireTime, isGuest } }

POST /api/auth/register
Body: { username, email, passwordHash, deviceId }
Response: { success, message }

POST /api/auth/login
Body: { username, passwordHash, deviceId }
Response: { success, data: { userId, username, email, token, tokenExpireTime, isGuest } }

POST /api/auth/forgot-password
Body: { email }
Response: { success, message }

POST /api/auth/verify
Headers: Authorization: Bearer <token>
Response: { success, data: { valid, userId } }
```

## 实现步骤

### 第一步：安装Addressables包
- Window → Package Manager → 搜索"Addressables" → 安装
- Window → Asset Management → Addressables → Groups → 创建默认配置

### 第二步：创建核心框架（优先级最高）
1. 创建目录结构（Scripts/Core, Scripts/Network, Scripts/HotUpdate, Scripts/Login, Scripts/UI, Scripts/Utils）
2. 实现工具类：
   - Crypto.cs（SHA256, AES加密）
   - JsonHelper.cs
   - Logger.cs
3. 实现核心管理器：
   - ConfigManager.cs（读取Resources/Config/下的JSON配置）
   - NetworkManager.cs（封装UnityWebRequest）
   - GameManager.cs（状态机和启动流程）

### 第三步：实现热更新系统
1. 创建配置文件：
   - Resources/Config/ServerConfig.json（服务器地址）
   - Resources/Config/AppVersion.json（当前版本号）
2. 实现热更新逻辑：
   - VersionChecker.cs（版本检查）
   - ResourceDownloader.cs（Addressables下载）
   - HotUpdateManager.cs（协调流程）
3. 创建HotUpdateUI.cs和HotUpdatePanel.prefab（显示下载进度）

### 第四步：实现登录系统
1. 实现数据模型：
   - UserData.cs（用户数据结构）
   - 定义LoginRequest/LoginResponse等数据结构
2. 实现认证逻辑：
   - TokenManager.cs（Token加密存储）
   - AuthService.cs（API调用封装）
   - LoginManager.cs（登录流程控制）
3. 创建UI：
   - LoginUI.cs + LoginPanel.prefab（游客/账号登录）
   - RegisterUI.cs + RegisterPanel.prefab（注册）
   - ForgotPasswordUI.cs + ForgotPasswordPanel.prefab（找回密码）

### 第五步：创建场景
1. 创建Launcher.unity场景：
   - 添加GameManager对象（挂载GameManager.cs）
   - 添加HotUpdatePanel
2. 创建Login.unity场景：
   - 添加LoginPanel, RegisterPanel, ForgotPasswordPanel
   - 添加Canvas和EventSystem
3. 配置Build Settings：
   - 场景顺序：Launcher → Login → SampleScene

### 第六步：集成测试
1. 测试启动流程（Launcher → 热更新检查 → Login）
2. 测试热更新（模拟有更新的情况）
3. 测试登录流程（游客登录、账号登录、记住密码）
4. 测试错误处理（网络断开、登录失败等）

## 安全考虑

### 客户端
- 密码永不明文存储，传输前SHA256哈希
- Token使用AES加密存储在PlayerPrefs
- 记住密码功能使用AES加密密码

### 服务器（需要后端开发者实现）
- 密码使用bcrypt加盐哈希存储
- Token使用JWT标准，设置过期时间
- API使用HTTPS加密传输
- 限制登录尝试次数，防止暴力破解

## 验证计划

### 功能验证
1. 启动游戏，观察Launcher场景是否正确初始化
2. 检查热更新流程：
   - 无更新时直接进入登录
   - 有更新时显示下载进度，完成后进入登录
3. 测试游客登录：点击游客登录按钮，成功后进入游戏主场景
4. 测试账号注册：填写信息，提交注册，验证服务器响应
5. 测试账号登录：输入账号密码，勾选记住密码，登录成功后进入游戏
6. 测试自动登录：关闭游戏重新打开，验证是否自动登录
7. 测试找回密码：输入邮箱，验证服务器发送邮件

### 异常验证
1. 断开网络，验证错误提示和重试机制
2. 输入错误密码，验证错误提示
3. Token过期，验证自动重新登录

### 性能验证
1. 使用Unity Profiler检查内存占用
2. 测试资源加载速度
3. 测试UI响应速度
